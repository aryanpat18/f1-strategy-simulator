📘 Project Summary: F1 Strategy Intelligence Platform
What is this project?
This project is a Formula 1 race strategy intelligence system.
In simple terms, it is a tool that:
collects historical Formula 1 race data
learns patterns from past races
predicts what might happen in a future race
simulates thousands of “what-if” scenarios
and recommends the best race strategy for a driver
It works before, during, and after a race.
Think of it as a digital race strategist.
Why does this exist?
In Formula 1, races are often won or lost because of:
when a driver pits
which tires they use
traffic and dirty air
safety cars
tire degradation
risk vs reward decisions
Teams use complex software to answer questions like:
Should we pit now or later?
Is a one-stop or two-stop strategy better today?
How risky is this decision?
What would have happened if we did something else?
This project recreates that kind of decision-making system, using real F1 data and modern machine learning — but built locally and openly.
High-Level Architecture (Big Picture)
The system has five major parts working together:
Data Pipeline – gets F1 data into a database
Machine Learning Models – learn race behavior
Simulation Engine – runs thousands of race scenarios
API Backend – serves predictions and simulations
Interactive Dashboard – shows results visually
All of this runs locally using Docker, but is designed like a cloud system.
1️⃣ Data Pipeline (The Foundation)
What it does
Automatically downloads F1 race data using the FastF1 library
Stores it in a PostgreSQL database
Runs on a schedule using Apache Airflow
Validates data quality (no missing or impossible values)
What kind of data?
Lap times
Tire compounds
Pit stops
Weather
Driver positions
Telemetry (where available)
Why it matters
Without clean, structured data, nothing else works.
This pipeline ensures:
“We can re-run the entire system for any race, any year, reliably.”
2️⃣ Machine Learning Models (The Brain)
The system uses multiple ML models, each with a clear purpose.
A. Lap Time Prediction Model (Core Model)
What it does:
Predicts how fast a driver will complete a lap.
Inputs include:
Driver
Track
Lap number
Tire compound and tire age
Estimated fuel load
Track & air temperature
Whether the driver is stuck behind another car (dirty air)
Stint number
Outputs:
Not just one number, but a range:
optimistic lap time
most likely lap time
pessimistic lap time
This allows the system to reason about risk and uncertainty.
B. Tire Degradation Model
What it does:
Learns how different tires slow down over time on different tracks.
This helps explain why lap times change and improves strategy realism.
C. Pre-Race Strategy Prediction Model
What it does (before the race starts):
Looks at similar races from the past
Groups races with similar conditions (track, temperature, safety cars)
Predicts:
likely number of pit stops
expected stint lengths
best starting tire
probabilities of different strategies (e.g. 1-stop vs 2-stop)
This answers:
“Before lap 1, what strategies are most likely to work today?”
D. Winner & Top-5 Prediction Models
What they do:
Predict who is most likely to win the race
Predict which drivers are likely to finish in the top 5
They consider:
Driver’s past performance at that track
Recent form (last few races)
Current year car performance (cars change every year)
Reliability and DNFs
These predictions are probabilistic, not absolute.
3️⃣ Simulation Engine (The What-If Machine)
This is the heart of the system.
What it does
For a selected:
Grand Prix
Year
Driver
The engine:
Automatically generates many valid race strategies
Ensures F1 rules are followed
at least two different dry tire compounds must be used (unless wet)
Runs hundreds or thousands of simulated races
Each simulation uses ML-predicted lap times + randomness
Two Modes
Manual Mode
User specifies pit laps and tire choices manually.
Good for:
experimentation
learning
exploration
Automated Mode (Main Vision)
User simply selects:
GP
driver
number of simulations
The system:
generates strategies
simulates them
ranks them by performance and risk
Outputs
For each strategy:
average race time
best / worst case
probability of being faster than baseline
risk level
This answers:
“What is the best strategy — and how risky is it?”
4️⃣ Post-Race Learning Loop (Continuous Improvement)
After a real race finishes:
The system compares:
what the model predicted
what actually happened
Stores the errors
Detects patterns (e.g. model struggles in hot races)
Adjusts future predictions
This is how production ML systems improve over time.
It turns the project from:
“a one-off model”
into
a learning system
5️⃣ Strategy Optimization (Prescriptive Intelligence)
Instead of just testing ideas, the system can now recommend actions.
It uses optimization techniques to:
minimize total race time
balance risk vs reward
respect constraints (tires, pit loss, race length)
This answers:
“What should we do?”
6️⃣ LLM-Powered GP Summaries (Human Understanding)
For any Grand Prix, the system can generate:
a one-page summary of the last 10 years
key track characteristics
strategic insights
The language model does not invent facts — it explains insights derived from data.
This helps humans understand why the system recommends something.
7️⃣ User Interface (How Humans Use It)
The Streamlit dashboard allows users to:
select a race and driver
view predictions
run simulations
explore strategy trade-offs visually
see charts, distributions, and comparisons
It turns complex ML outputs into decision-friendly visuals.
